SERVIDORES DE BASE DE DATOS

Un servidor de base de datos o database server, es un tipo de software de servidor que permiten la organización de la información usando tablas, índices y registros. Es un sistema bajo arquitectura cliente/servidor que proporciona servicios de gestión, administración y protección de la información a través de conexiones de red. Cuentan con protocolos definidos, los usuarios acceden a través de aplicaciones clientes (herramientas del propio sistema como aplicaciones de terceros). Se especializa en servir consultas a clientes remotos o locales que solicitan información o desean realizar modificaciones a los registros y tablas que existen dentro del sistema. Las bases de datos sirven para gestionar y administrar inmensas cantidades de información, lo cual es fundamental para empresas, instituciones, universidades o bancos, que almacenan datos de usuarios/clientes: emails, direcciones, teléfonos, ingresos y egresos, calificaciones, etc.

Funciones:

Provee información a otras aplicaciones web o equipos/hosts. El modelo cliente servidor permite acceder a la información que se guarda en las diferentes bases de datos. Cuando el cliente accede usando un usuario, contraseña y nombre de host, se le permite realizar diferentes tareas dependiendo de los privilegios que posea. Hay usuarios con diferentes tipos de privilegios, como el de administrador que tiene permitido administrar por completo las bases de datos a las que se conecta, en cambio, otros usuarios tienen privilegios parciales como solo leer datos. En todo sistema abierto, debe proporcionarse un potente mecanismo de seguridad que garantice que ningún intruso pueda acceder o corromper la integridad del sistema.

Características:

  • Posee una consola de gestión de la base de datos.
  • Proporciona la posibilidad de administración de documentos, registros, todo lo relacionado con los buscadores de Google, Mozilla y Explorer.
  • Aplicación Web de administración para el acceso restringido desde cualquier localización.
  • Gestiona servidores de correos electrónicos y el contenido de las diversas páginas web existentes.
  • Se usan para las páginas web con contenido dinámico, alojando todos los datos, elementos e información de la misma.
  • Mantienen un diseño clásico y se encuentran repletas de información relevante y útil para usuarios de la red, almacenando los datos de los usuarios y los movimientos que hagan dentro de la web.
  • Acceso JDBC para conectar desde aplicaciones Java, applets, etc.

Usos Populares:

Existen muchos usos populares, ya sea el uso de software para registros médicos, organización de documentos de textos, estudios contables, pago de impuestos, modificar aspectos de sitios Web, y más.

Algunos ejemplos son:

MySQL server:

  • Es un servidor de bases de datos de tipo relacional, considerada la más popular.
  • Se utiliza en desarrollo web y aplicaciones que utilizan lenguajes populares como PHP, Ruby o Python.
  • Brinda acceso directamente en el servidor o a los computadores remotos mediante usuario y contraseña.
  • Se caracteriza por la ejecución de tareas en simultáneo tanto lectura como escritura.
  • Ofrece gran velocidad de acceso a los datos.
  • Soporta múltiples motores de almacenamiento.
  • Permite uso de índices, múltiples transacciones, balanceo de carga, clustering, entre otras.

PostgreSQL server:

  • Es un servidor de tipo relacional, open source y orientado a objetos.
  • Es una de las opciones más elegidas, generalmente en entornos empresariales y gubernamentales.
  • Se caracteriza por ofrecer una gran estabilidad, robustez y velocidad a la hora de administrar datos.
  • Posee una excelente forma de manejar grandes volúmenes de datos y alta simultaneidad de usuarios

Microsoft SQL Server:

  • Es el servidor de base de datos SQL relacional de Microsoft.
  • Muy popular entre usuarios de la plataforma Windows Server.
  • Ofrece una compatibilidad nativa con el lenguaje de programación ASP/ASP.NET, así como con toda la suite de desarrollo de aplicaciones de sistemas operativos de Windows.
  • Cuenta con un soporte de procedimientos almacenados, transacciones, administración mediante una interfaz gráfica.
  • Da la opción del uso de comandos DML y DDL, uso del servidor remoto, y también ofrece una interfaz de comandos llamada oSQL o SQLCMD.
